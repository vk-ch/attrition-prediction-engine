# Employee Attrition Prediction Engine

## The Business Problem

Employee attrition costs companies between 50% and 200% of an employee's annual salary in replacement costs. At a 16% attrition rate, a 1,000-person company loses roughly 160 people per year — and most of it is preventable if you know who is at risk and why, early enough to act.

This project builds a machine learning model to predict which employees are most likely to leave, explains *why* using SHAP values, and packages those insights into a format an HR Business Partner can actually use. The goal is not just model accuracy — it's actionable recommendations tied to specific retention levers.

## What I Built

- **SQL-first EDA** using DuckDB to explore hiring funnels, attrition patterns by department/role/tenure, and cross-segment risk signals — the same workflow as a Snowflake or Redshift environment
- **Logistic Regression** (statsmodels) with odds ratios and p-values to identify statistically significant attrition drivers
- **Random Forest** classifier with `class_weight='balanced'` to handle the 84/16 class imbalance, achieving **AUC 0.87** on the test set
- **SHAP explainability** — beeswarm summary, dependence plots, and individual waterfall explanations for high-risk employees
- **Risk tiering dashboard** — segmenting employees into High / Medium / Low risk and mapping concentration by department and role
- **Business memo** in `outputs/business_memo.md` — findings written for a CPO, not a data scientist

## Key Findings

- Overtime is the strongest single attrition predictor — employees on overtime leave at 2-3x the rate of comparable colleagues, compounding sharply when job satisfaction is low
- The highest-risk tenure window is years 1-2. Employees in this window leave at nearly 3x the rate of 6+ year employees, pointing to onboarding and manager quality as the primary lever
- A small "triple-risk" group — overtime + low satisfaction + early tenure — leaves at ~60%. In a 1,500-person company this is typically 20-50 people. Identifying them monthly is the highest-ROI retention action available
- The model is 9x more accurate at identifying future leavers than random selection

## How to Run

**1. Clone the repo**
```bash
git clone https://github.com/YOUR_USERNAME/attrition-prediction-engine.git
cd attrition-prediction-engine
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Open the notebook**
```bash
jupyter notebook notebooks/01_attrition_analysis.ipynb
```

The notebook downloads the dataset automatically from a public URL. If that fails, see `data/README.md` for the Kaggle download link.

**4. Run all cells top-to-bottom**

All 13 output charts save automatically to the `outputs/` folder.

## Methodology

| Step | Approach |
|------|----------|
| Data loading | pandas + DuckDB (SQL-first workflow) |
| EDA | 7 SQL queries covering department, age, tenure, satisfaction, role |
| Feature engineering | 3 derived features in SQL: satisfaction composite, stalled promotion flag, manager tenure gap |
| Model 1 | Logistic Regression (statsmodels) — interpretable coefficients + p-values |
| Model 2 | Random Forest (scikit-learn) — AUC 0.87, balanced class weights |
| Explainability | SHAP TreeExplainer — summary, dependence, and waterfall plots |
| Risk scoring | Probability thresholds → High / Medium / Low tiers |

## Dataset

**IBM HR Analytics Employee Attrition & Performance**  
1,470 employees, 35 features, 16.1% attrition rate  
Source: [Kaggle](https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset)
